```

├── max_concurrent_papers = 10
└── 每篇论文在独立线程中运行

内层并发（每篇论文内部）
├── 5个分析器同时运行
├── max_concurrent_analyzers = 5
└── 使用 DeepSeek API 并发调用

总并发能力: 10 × 5 = 50 个同时请求
```

## 🚀 使用方法

### 启动 GUI

```bash
cd LiteratureHub

# 方式1：双击启动（Windows）
启动GUI.bat

# 方式2：命令行启动
python launch_gui.py

# 方式3：直接运行
python scripts/page1_gui.py
```

### GUI 使用流程

1. **Page 1（文献管理）**：
   - Elsevier 搜索 → 搜索学术文献
   - SciHub 下载 → 自动下载 PDF
   - MinerU 转换 → PDF 转 Markdown

2. **切换到 Page 2（AI 分析）**：
   - 点击 "🧠 AI 分析" 标签页
   - 查看已转换的 Markdown 文献列表
   - 设置并发参数（外层10、内层5）
   - 点击 "▶️ 开始分析"

3. **分析进度监控**：
   - 查看总体进度条
   - 查看当前论文状态
   - 查看5个分析器的实时状态
   - 查看日志输出

## ⚙️ 配置说明

### API 密钥配置

**位置**: `config/api_keys.yaml`

```yaml
# DeepSeek API（Page 2 AI 分析专用 - 无限并发）
deepseek:
  api_key: "YOUR_DEEPSEEK_API_KEY"
  base_url: "https://api.deepseek.com"
  model: "deepseek-chat"  # 或 deepseek-reasoner
  timeout: 300

  # 并发配置
  max_concurrent: 100  # 无限并发
  request_timeout: 300
```

### 工作流配置

**位置**: `config/workflow.yaml`

```yaml
# Page 2 分析配置（双层并发）
analysis_v2:
  # 并发配置
  concurrency:
    outer_layer: 10  # 外层并发：同时处理的论文数
    inner_layer: 5   # 内层并发：每篇论文的分析器数
    max_total: 50    # 理论最大并发数

  # DeepSeek API 配置
  api:
    provider: "deepseek"
    model: "deepseek-chat"  # 或 deepseek-reasoner
    timeout: 300

  # 输出目录
  output_dir: "data/agent_results"
```
