# LiteratureHub 数据资产与信息层级指南

## 概述

本文档说明 LiteratureHub 项目中**博士研究方向调研汇报**的数据资产结构和信息流转过程。

## 应用场景

**用户**: 博一刚入学的研究生
**任务**: 对"大型风力机气动"领域进行调研
**目标**:
- 向导师汇报调研结果
- 找到研究空白和机会点
- 确定具体的研究方向
- 制定可行的研究计划
- **说服导师同意研究方向**

## 数据使用指南

### 用于 PPT 内容生成
**推荐路径**:
1. 从 `part1-4` Markdown 文件提取内容
2. 按 10 页结构组织
3. 参考 HTML 格式规范
4. 生成最终 HTML 文件

**数据来源**:
- Page 1-3 → `part1_literature_review.md`
- Page 4-6 → `part2_innovation_points.md`
- Page 7-8 → `part3_methodology.md`
- Page 9-10 → `part4_future_work.md`

### 用于深度研究
**推荐路径**:
1. 查看 `phase1_overview.json` 了解全貌
2. 阅读目标领域的 `domain_analysis.json`
3. 查阅 `agent_results/` 中的具体文献分析
4. 必要时回到 `full.md` 阅读原文

### 用于文献检索
**推荐路径**:
1. 从高影响力文献列表入手
2. 按领域筛选目标文献
3. 阅读 `innovation.json` 了解创新点
4. 决定是否深入阅读 PDF

## 数据流转过程

### 分析阶段（已完成）
1. **PDF → Markdown**: MinerU 自动解析
2. **Markdown → Agent Results**: Claude Code Agent 逐篇分析
3. **Agent Results → Phase 1-2**: Claude Code Agent 综合提炼
4. **Phase 1-2 → Part 1-4**: Claude Code Agent 总结生成

### 应用阶段（进行中）
5. **Part 1-4 → HTML 10 页**: 按本指南生成 PPT 内容
6. **HTML → PPT**: 用户手动转换（使用工具如 Pandoc）

## 维护建议

### 数据更新
- 新增文献时，重复 Level 0 → Level 2 流程
- 更新 Phase 1-2 概览分析
- 重新生成 Part 1-4 总结

### 质量控制
- 定期检查 Agent 分析结果准确性
- 验证领域分类一致性
- 更新高影响力文献列表

### 版本管理
- 为每次分析生成打标签/版本号
- 保留历史版本便于回溯
- 记录数据处理日志
